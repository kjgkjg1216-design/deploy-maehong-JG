# 매홍 L&F 통합 재고 관리 챗봇

Flask + GPT-4o RAG 기반의 사내 통합 데이터 조회 챗봇입니다.
재고, 발주, 생산, BOM, 매출/출하, 입고, Monday.com 업무 데이터를 자연어로 질의할 수 있습니다.

---

## 시스템 구성

```
┌─────────────────────────────────────────────────────┐
│                    사용자 (브라우저)                     │
│   챗봇 (/)  │  관리자 (/admin)  │  업로드 (/upload)    │
└──────────────────────┬──────────────────────────────┘
                       │
              ┌────────▼────────┐
              │   Flask 서버     │
              │   (app.py)      │
              │   port 5000     │
              └───┬────┬────┬───┘
                  │    │    │
     ┌────────────┤    │    ├────────────┐
     ▼            ▼    │    ▼            ▼
  엑셀 CSV     GPT-4o  │  아마란스10   Monday.com
  (재고/단가)   (RAG)   │  ERP API     API
                       │
                  로컬 CSV
               (data/ 폴더)
```

### 데이터 소스

| 소스 | 데이터 | 수집 방법 |
|------|--------|----------|
| 엑셀 파일 | 외주 재고일지, 자사 부자재 재고, 단가, 부자재 규격 | `convert_*.py` → CSV 변환 |
| 아마란스10 ERP | 구매발주, 외주발주, 생산지시, 생산실적, 입고, 출하, 출고, BOM | `fetch_all.py` → API 수집 |
| Monday.com | 전사 공지사항, 업무관리, 부자재 규격 | `fetch_monday.py` → API 수집 |

### 주요 파일

| 파일 | 설명 |
|------|------|
| `app.py` | Flask 메인 앱 (RAG 검색 + GPT 응답 + HTML UI) |
| `amaranth_api.py` | 아마란스10 API 공통 인증 모듈 |
| `fetch_all.py` | 아마란스 전체 데이터 일괄 수집 |
| `fetch_bom.py` | BOM(자재명세서) 수집 (일반 + 외주BOM) |
| `fetch_orders.py` | 구매발주 + 외주발주 수집 |
| `fetch_production.py` | 생산지시 + 생산실적 수집 |
| `fetch_sales.py` | 출하(매출) + 출고 수집 |
| `fetch_monday.py` | Monday.com 보드 데이터 수집 |
| `convert_excel.py` | 외주 재고일지 엑셀 → CSV |
| `convert_price.py` | 단가 엑셀 → CSV |
| `convert_jasa.py` | 자사 부자재 재고 엑셀 → CSV |
| `convert_spec.py` | 부자재 규격 엑셀 → CSV |
| `run_chatbot.bat` | 데이터 변환 + 서버 시작 일괄 실행 |

## 설치 및 실행

### 1. 환경 준비

```bash
# Python 3.10+ 필요
pip install -r requirements.txt
```

### 2. 환경변수 설정

`.env.example`을 `.env`로 복사 후 실제 키 입력:

```bash
cp .env.example .env
```

```env
OPENAI_API_KEY=sk-xxxx
AMARANTH_ACCESS_TOKEN=xxxx
AMARANTH_HASH_KEY=xxxx
AMARANTH_GROUP_SEQ=gcmsAmaranthXXXXX
AMARANTH_CALLER_NAME=API_gcmsAmaranthXXXXX
MONDAY_API_KEy=xxxx
```

### 3. 데이터 준비

```bash
# 엑셀 파일을 프로젝트 루트에 배치 후:
python convert_excel.py    # 외주 재고일지 → data/YYYYMMDD_재고일지.csv
python convert_price.py    # 단가 → data/YYYYMMDD_단가.csv
python convert_jasa.py     # 자사 부자재 재고 → data/YYYYMMDD_자사재고.csv
python convert_spec.py     # 부자재 규격 → data/YYYYMMDD_부자재규격.csv

# 아마란스 ERP 데이터 수집 (25년6월~현재)
python fetch_all.py

# Monday.com 데이터 수집
python fetch_monday.py
```

### 4. 서버 실행

```bash
python app.py
# 또는 일괄 실행:
run_chatbot.bat
```

접속: `http://localhost:5000`

## 주요 기능

### 챗봇 (/)

자연어로 질문하면 GPT-4o가 RAG 방식으로 데이터를 검색하여 응답합니다.

#### 재고 조회

| 질문 예시 | 설명 |
|----------|------|
| `C0018 재고수량 알려줘` | 품번별 자사+외주 통합 재고 조회 |
| `오트밀 부재료 재고` | 품명 검색으로 부재료 재고 조회 |
| `더고은 부재료 재고` | 외주처별 부재료 재고 |
| `자사 롯데 부재료 재고` | 자사 재고에서 롯데마트 관련 검색 |
| `외주업체별 재고비용` | 외주처별 재고금액 집계 |

#### 발주/입고 조회

| 질문 예시 | 설명 |
|----------|------|
| `3월5일 발주현황` | 특정 날짜 구매발주 조회 |
| `25년12월 외주발주현황` | 과거 월별 외주발주 조회 |
| `조운정미 발주현황` | 거래처별 발주 필터링 |
| `C0018 입고내역` | 품번별 입고 이력 |
| `3월 입고현황` | 월별 입고 조회 |

#### 생산계획/실적 조회

| 질문 예시 | 설명 |
|----------|------|
| `3월24일 생산계획` | 특정일 생산지시 조회 (아마란스) |
| `G0010 생산계획` | 품번별 생산지시 이력 |
| `3월24일 생산계획 대비 실적` | 생산지시 vs 실적 달성률 비교 |
| `3월 생산실적` | 월별 생산실적 조회 |

#### BOM 조회

| 질문 예시 | 설명 |
|----------|------|
| `G0010 BOM 알려줘` | 자사제품 BOM 조회 |
| `H0270 BOM` | 외주제품 BOM 조회 (외주BOM 포함) |
| `E0009 BOM` | 반제품 BOM 조회 |
| `G0010 10000ea 생산 부재료 부족수량` | BOM 기반 소요량 vs 현재고 비교 |

#### 매출/출하 조회

| 질문 예시 | 설명 |
|----------|------|
| `3월 매출현황` | 월별 출하(매출) 조회 |
| `B0340 판매량` | 품번별 판매 이력 |
| `청통본가 출하내역` | 거래처별 출하 조회 |

#### Monday.com 조회

| 질문 예시 | 설명 |
|----------|------|
| `전사 공지사항 보여줘` | Monday.com 보드 검색 |
| `임직원 강점혁명 알려줘` | 보드 내용 조회 + 하위 아이템 |
| `SCM 회의 자료` | 보드명 검색 |

#### 기타

| 질문 예시 | 설명 |
|----------|------|
| `B0384 정보 알려줘` | 품번 통합 정보 (단가+규격+사이즈+재질+MOQ) |
| `원재료 단가 알려줘` | 단가 파일 기준 원재료 전체 단가 조회 |
| `제일산업에서 제조하는 부재료` | 부재료 제조업체별 규격 조회 |

### 관리자 대시보드 (/admin)

- 외주업체별 부재료/원재료 재고금액 집계
- 자사 부자재 재고 현황
- 원형 차트 (업체별 재고비율 %)
- 부재료 vs 원재료 금액 비교 바 차트
- 업체별 상세 테이블 (펼치기/접기)

### 파일 업로드 (/upload)

- 엑셀 파일 드래그 앤 드롭 업로드
- 동일 파일명 자동 교체
- 업로드 후 서버 재시작 시 반영

### 외주처별 재고 뷰

챗봇 상단 버튼 (데이웰즈, 더고은, 정성, 청통본가, 자사)으로 각 업체의 원재료/부재료 재고를 즉시 조회

## 품번 체계

| Prefix | 분류 | 데이터 소스 |
|--------|------|-----------|
| A | 원재료 | 단가 파일, 외주 재고일지 |
| B, C, D | 부재료 (파우치, 박스, 흡수제 등) | 재고일지, 자사재고, 부자재 규격 |
| E | 반제품 | BOM |
| F | 기타 반제품 | BOM |
| G | 자사 제품 | BOM, 생산지시/실적 |
| H | 외주 제품 (부자재 매홍) | 외주BOM, 생산지시/실적, 제품재고 |
| I | 외주 제품 (상품) | 발주, 외주BOM, 제품재고 |

## 아마란스10 API 연동

### 인증 프로토콜

```
Authorization: Bearer {ACCESS_TOKEN}
wehago-sign: Base64(HMAC-SHA256(HASH_KEY, ACCESS_TOKEN + transaction-id + timestamp + URL_PATH))
transaction-id: 32자리 랜덤 문자열
timestamp: Unix timestamp (초 단위)
groupSeq: 그룹시퀀스
CallerName: 호출 구분명
```

| 모듈 | API | 용도 |

## 기술 스택

- **Backend**: Python 3.12, Flask
- **AI**: OpenAI GPT-4o (RAG, temperature=0)
- **ERP**: 더존 아마란스10 Cloud API
- **협업툴**: Monday.com API
- **Frontend**: HTML/CSS/JS (인라인), Chart.js, marked.js
- **데이터**: pandas, CSV (UTF-8-BOM)

## 보안

- `.env` 파일은 `.gitignore`로 제외 (API 키 미노출)

- 데이터 파일(CSV, 엑셀)은 로컬 전용
